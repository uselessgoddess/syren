# Roadmap

syren's vision is a modern, eBPF-powered observability tool for syscalls — a
`strace` you'd actually enjoy using, with broad autogenerated coverage,
structured output, low overhead, and a live TUI. This document lays out how it
gets there in stages.

It is a direction, not a contract: priorities will shift with use and feedback.
Each milestone is meant to ship something useful on its own.

## [X] v0.1 — Foundation (shipped)

The tracing pipeline, end to end, on a zero-privilege backend.

- [x] Cargo workspace of six focused crates with a clean, acyclic dependency flow.
- [x] **Autogenerated syscall table** from the kernel's `syscall_64.tbl` +
      metadata, covering the full x86-64 surface (373 syscalls) with categories
      and argument shapes for the ~90 hot-path calls.
- [x] **Unprivileged `ptrace` backend**: spawn or attach, enter/exit pairing,
      follow-forks, exit-status propagation.
- [x] **Metadata-driven decoder** reading tracee memory for paths and buffers,
      with symbolic flag decoding and errno-aware return values.
- [x] **Output formats**: strace-compatible text, NDJSON, and a `-c` summary table.
- [x] **CLI** with syscall/category filtering (`-e`), and a strace-clean split of
      trace output (stderr) from the tracee's own streams.
- [x] **Efficient CI**: fmt, clippy (`-D warnings`), tests (default + all
      features), docs, MSRV, and supply-chain checks.

## [ ] v0.2 — The eBPF backend

syren's performance north star: trace without two context switches per syscall.

- [ ] aya program on the `raw_syscalls` `sys_enter`/`sys_exit` tracepoints.
- [ ] Per-CPU enter/exit pairing and a ring buffer to userspace, emitting the
      **same `Event` stream** the ptrace backend does (decode/output reused as-is).
- [ ] Graceful capability detection and fallback to ptrace when eBPF is
      unavailable (no `CAP_BPF`, no BTF).
- [ ] Overhead benchmarks (ptrace vs. eBPF) under load.

## [ ] v0.3 — Deeper decoding

Make the output as informative as strace's, then more so.

- [ ] Struct decoders for common arguments (`struct stat`, `sockaddr`, `timespec`,
      `iovec`, …) driven by metadata.
- [ ] fd → path resolution and annotation (`3</etc/passwd>`).
- [ ] Richer string/buffer handling: configurable length caps, abbreviation,
      non-UTF-8 escaping parity with strace.
- [ ] Per-syscall return-value interpretation (e.g. `mmap` address, `clone` pid).

## [ ] v0.4 — Live TUI

A `ratatui` interface for watching a process in real time.

- [ ] Live syscall stream with category colouring and pause/scroll.
- [ ] Live aggregation pane (top syscalls by count/time, error rates).
- [ ] Per-process / per-thread tree view for multi-threaded targets.
- [ ] Interactive filtering by name, category, fd, or pid.

## [ ] v0.5 — Export & integration

Make syren's data flow into the rest of the observability stack.

- [ ] **Chrome trace** / Perfetto JSON for flame-style timelines.
- [ ] **Parquet** export for offline analysis at scale.
- [ ] **Prometheus** metrics endpoint for per-syscall counters and latencies.
- [ ] Stable, versioned NDJSON schema.

## [ ] v0.6 — strace drop-in & scripting

Lower the switching cost and open an extension surface.

- [ ] **strace-compat mode**: accept strace's common flags and match its output
      closely enough to slot into existing scripts.
- [ ] Cost analysis: highlight the syscalls dominating wall-clock and the ones
      that erred.
- [ ] **Rhai scripting** hooks for custom filtering, aggregation and alerting on
      the event stream.

## Beyond

- [ ] Additional architectures (`aarch64`) via per-arch generated tables.
- [ ] Container/namespace awareness.
- [ ] `seccomp`-assisted selective tracing to cut overhead further.
- [ ] Packaging: crates.io release, prebuilt binaries, distro packages.
